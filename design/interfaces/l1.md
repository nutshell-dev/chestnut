## FileSystem [capability, direct + DI]

**生产方**：`l1_filesystem`

**消费方**：几乎所有模块（DI 加 direct）

**接口签名**：

```ts
export interface FileSystemOptions {
  baseDir: string;
}

export interface FileSystem {
  // 异步 API
  read(path: string): Promise<string>;
  writeAtomic(path: string, content: string): Promise<void>;
  append(path: string, content: string): Promise<void>;
  delete(path: string): Promise<void>;
  move(fromPath: string, toPath: string): Promise<void>;
  exists(path: string): Promise<boolean>;
  isDirectory(path: string): Promise<boolean>;
  stat(path: string): Promise<StatInfo>;
  list(path: string, options?: ListOptions): Promise<FileEntry[]>;
  ensureDir(path: string): Promise<void>;
  removeDir(path: string): Promise<void>;

  // 同步 API（hot path / lockfile / 高频写）
  readSync(path: string): string;
  writeAtomicSync(path: string, content: string): void;
  writeExclusiveSync(path: string, content: string): void;  // EEXIST 不可改
  appendSync(path: string, content: string): void;
  readBytesSync(path: string, start: number, end: number): Buffer;
  statSync(path: string): StatInfo;
  existsSync(path: string): boolean;
  moveSync(fromPath: string, toPath: string): void;
  deleteSync(path: string): void;
  ensureDirSync(path: string): void;
  listSync(path: string, options?: ListOptions): FileEntry[];

  // 路径解析
  resolve(relativePath: string): string;
}

export interface ListOptions {
  recursive?: boolean;
  includeDirs?: boolean;
  pattern?: string;  // glob
}

export interface StatInfo {
  size: number;
  mtime: Date;
  ctime: Date;
  isDirectory: boolean;
  isFile: boolean;
}

export interface FileEntry {
  name: string;
  path: string;
  isDirectory: boolean;
  isFile: boolean;
  size: number;
  mtime: Date;
}

export class FileNotFoundError extends Error {
  readonly path: string;
}

export const IGNORE_PATTERN: string;  // '.tmp_' / temp 残片命名前缀（type-level 资源 / 由 Assembly cleanup 引用）
```

**使用语义**：

- 目标不存在抛 FileNotFoundError
- writeExclusiveSync 目标已存在抛 EEXIST（PID 锁实现依据，不可改为「覆盖」语义）
- 磁盘满、权限不足等 OS 错误原样抛
- 路径相对 baseDir 解析（baseDir 构造期传入，运行时不变）
- 跨进程并发不保证（多进程写同一文件由调用方协调）
- 原子写通过 temp + rename 实现，rename 前崩溃不污染原文件 / 残片命名前缀 = `IGNORE_PATTERN` / 启动期残片清理归 L6 Assembly

**归本模块**：clawforum 文件 I/O 能力的唯一入口。业务模块要做文件操作必经本模块。

**不归本模块**：

- 业务目录语义（具体业务目录加文件名规则），归各业务模块自治
- 权限策略（agent 能不能读某路径），归 caller 自治（caller 在 call FileSystem 前自己 check / FileSystem 0 知权限业务）
- 越界守护（业务 sandbox 能力），归 caller 自治。注：base-dir traversal 守护（normalized.startsWith('..') + symlink）= L1 OS 合规保留
- disk artifacts 聚合（如 gitignore content），归 L6 Assembly 装配期组装
- 协议解析（frontmatter 加 JSON 加 YAML 等格式），归调用方
- 文件变化通知（watch），归 L1 FileWatcher

**不可消除耦合理由**：M#3 资源唯一归属（OS 文件系统能力是单一资源）加 M#5 业务模块不允许直接 import `node:fs` 加 `node:path`。Design Principle「clawforum 支持分布式部署和跨 OS 平台」derive 必经本模块抽象。

## ProcessExec [capability, direct + DI]

**生产方**：`l1_process_exec`

**消费方**：

- `l2_process_manager`（direct / 进程生命周期编排 spawn 加 isAlive）
- `l2_snapshot`（direct / git exec 子进程）
- `l2_command_tool`（DI / agent exec 工具）
- `l4_contract_system`（direct / 脚本 acceptance 子进程）

**接口签名**（应然 / per arch §2 表 2「exec / detached spawn / 进程停止信号 / 存活检查 / pattern 查找」5 能力 / phase439 加 `spawnDetached` + phase 495 加 `kill` + `isAlive` + `findByPattern` + Signal type + ProcessInfo type / pgrepSync deprecated）：

```ts
export interface ExecOptions {
  cwd: string;                                       // working directory (required)
  timeout?: number;                                  // ms, clamped to [MIN, MAX]
  signal?: AbortSignal;
}

export interface ExecResult {
  output: string;       // stdout + stderr 时序合并单字段（phase 483 / D5「信息不丢失」derive / 时序保留）
  exitCode: number;
}

export interface SpawnDetachedOptions {
  cwd?: string;
  env?: Record<string, string | undefined>;
  logFile?: string;       // 内部 open/close fd 包装
}

export interface ProcessInfo {
  pid: number;
  command: string;
}

export type Signal = 'TERM' | 'KILL' | 'INT';        // 中性 signal 名 / 内部按 OS 分支映射

// 5 应然方法（强制 cmd + args[] 形态 / 不暴露 shell mode / kill+isAlive+findByPattern sync 形态 phase 495 reframe）

export function exec(cmd: string, args: ReadonlyArray<string>, options: ExecOptions): Promise<ExecResult>;
export function spawnDetached(cmd: string, args: ReadonlyArray<string>, options: SpawnDetachedOptions): { pid: number };
export function kill(pid: number, signal: Signal): void;                    // phase 495 reframe sync (β / OS call 本身 sync)
export function isAlive(pid: number): boolean;                              // phase 495 reframe sync
export function findByPattern(pattern: string): ProcessInfo[];              // phase 495 reframe sync (替 pgrepSync / pgrepSync deprecated)

// timeout 边界常量
export const PROCESS_EXEC_TIMEOUT_MIN_MS: number;
export const PROCESS_EXEC_TIMEOUT_MAX_MS: number;
export const PROCESS_EXEC_DEFAULT_TIMEOUT_MS: number;

export class ProcessExecError extends Error {
  readonly output: string;                           // stdout + stderr 时序合并（phase 483 / 单字段 align ExecResult）
  readonly exitCode: number | null;
  readonly code?: string;                            // OS 错误代码（如 ENOENT）
  readonly signal?: string;                          // 终止信号
  readonly killed: boolean;                          // 进程是否被 kill 终止
  readonly maxBufferExceeded: boolean;
}
```

**归本模块**：clawforum 进程能力的唯一入口。业务模块要执行 OS 进程必经本模块。

**不归本模块**：

- 进程业务编排（哪个进程是 daemon、PID 文件管理、孤儿清理等），归 L2 ProcessManager
- shell 兼容层（`sh -c` 跨 OS 不一致），调用方自负风险
- 命令白黑名单，归 L2 CommandTool 加 L6 Assembly 装配期决定

**不可消除耦合理由**：M#3 资源唯一归属（OS 进程能力是单一资源）加 M#5 业务模块不允许直接 import `node:child_process`。Design Principle「clawforum 支持分布式部署和跨 OS 平台」derive 必经本模块抽象。

## LLMProvider [capability, DI]

**生产方**：`l1_llm_provider`

**消费方**：

- `l2_llm_orchestrator`（DI，唯一调用方）

**接口签名**：

```ts
// LLM 协议层 message types（本模块 own / provider 异构吸收的一部分）

export interface Message {
  role: 'user' | 'assistant' | 'system' | 'tool';
  content: string | Array<TextBlock | ToolUseBlock | ToolResultBlock>;
}

export interface TextBlock {
  type: 'text';
  text: string;
}

export interface ToolUseBlock {
  type: 'tool_use';
  id: string;
  name: string;
  input: Record<string, unknown>;
}

export interface ToolResultBlock {
  type: 'tool_result';
  tool_use_id: string;
  content: string;
  is_error?: boolean;
}

// LLM API call IO type（本模块 own / 单 provider call 的 input/output schema）

export interface LLMCallOptions {
  messages: Message[];
  system?: string;
  tools?: ToolDefinition[];
  maxTokens?: number;
  temperature?: number;
  model?: string;          // override per call
  timeoutMs?: number;
  signal?: AbortSignal;    // user abort + step_yield only (idle timeout 由 L2 LLMOrchestrator 编排)
  idleTimeoutMs?: number;  // per-attempt idle timeout (0 / undefined = 禁用)
}

export interface LLMResponse {
  content: Array<TextBlock | ToolUseBlock>;
  stopReason: 'end_turn' | 'max_tokens' | 'tool_use' | 'stop_sequence' | string;
  usage: { inputTokens: number; outputTokens: number };
  model: string;
}

export interface StreamChunk {
  type: 'text_delta' | 'thinking_delta' | 'thinking_signature' | 'tool_use_start' | 'tool_use_delta' | 'done' | 'reset' | 'provider_failed';
  delta?: string;
  toolUse?: { id: string; name: string; partialInput?: string };
  usage?: { inputTokens: number; outputTokens: number };
  signature?: string;       // anthropic extended thinking
  stopReason?: string;      // type='done'
  provider?: string;        // type='reset' | 'provider_failed'
  timeoutMs?: number;       // type='reset'
  error?: string;           // type='provider_failed'
  model?: string;           // type='provider_failed'
}

// LLM API tool spec（喂给 LLM 的 tool definition / 跟 L2 ToolProtocol.Tool 区分：Tool 是 clawforum 内部 / ToolDefinition 是 LLM API 接受的 schema）

export interface ToolDefinition {
  name: string;
  description: string;
  input_schema: Record<string, unknown>;  // JSON Schema
}

// Provider 接口（单 provider call / 不持容错语义）

export interface LLMProvider {
  readonly name: string;
  readonly model: string;
  call(options: LLMCallOptions): Promise<LLMResponse>;
  stream?(options: LLMCallOptions): AsyncIterableIterator<StreamChunk>;
}

export class LLMProviderError extends Error {
  readonly code: 'network' | 'provider_error' | 'invalid_request' | 'unknown';
  readonly provider: string;
}
```

注：本模块 own LLM 协议层 message + IO + tool definition type 单源（provider 异构吸收的一部分）/ L2 LLMOrchestrator + L2 DialogStore + L3 StepExecutor 等都 import 本模块 type。健康探测加 abort 编排归 L2 LLMOrchestrator（详该节 healthCheck）/ 本模块单 provider call 不持容错语义。

**使用语义**：

- 网络错误、provider 服务错误抛 LLMProviderError
- 不做重试加 failover（容错归 L2 LLMOrchestrator）
- 不做健康探测加 abort 编排（归 L2 LLMOrchestrator）
- 一个 provider 一个实例（多 provider 协调归 L2 LLMOrchestrator）
- stream 为 optional（部分 provider 不支持流式）
- KV cache 标记由调用方决定（节约成本），本模块透传给 provider

**归本模块**：clawforum 单一 LLM provider 调用能力的唯一入口。业务模块要调 LLM 必经本模块（一般通过 L2 LLMOrchestrator 间接调用）。

**不归本模块**：

- 业务编排（重试、failover、circuit breaker、idle timeout），归 L2 LLMOrchestrator
- 多 provider 协调（primary 加 fallbacks 选择），归 L2 LLMOrchestrator
- 失败审计加业务事件，归 L2 LLMOrchestrator
- dialog 状态管理（messages 数组持久化），归 L2 DialogStore

**不可消除耦合理由**：M#3 资源唯一归属（单一 LLM provider 接入是单一资源）加 M#5 业务模块不允许直接调 provider 接口。Design Principle「clawforum 支持分布式部署和跨 OS 平台」derive 必经本模块抽象（provider 异构吸收）。

## Transport [capability, DI]

**生产方**：`l1_transport`

**消费方**：

- `l5_gateway`（DI，唯一上游加客户端实时交互桥接）

**接口签名**：

```ts
export interface TransportOptions {
  socketPath?: string;
}

export interface Connection {
  id: string;
  connectedAt: number;  // unix timestamp ms（实然 ms 而非 Date instance / 序列化友好）
}

export interface BroadcastFailure {
  connectionId: string;
  error: Error;
}

export type TransportErrorEvent =
  | { kind: 'callback_error'; callbackName: 'onConnect' | 'onDisconnect' | 'onMessage' | 'onTransportError'; connectionId?: string; error: Error }
  | { kind: 'server_error'; error: Error };

export interface Transport {
  listen(options?: TransportOptions): Promise<void>;
  close(): Promise<void>;
  send(connectionId: string, data: string): void;
  broadcast(data: string): { failed: BroadcastFailure[] };
  getConnections(): Connection[];
  onConnect(cb: (conn: Connection) => void): void;
  onDisconnect(cb: (conn: Connection, reason?: Error) => void): void;
  onMessage(cb: (conn: Connection, data: string) => void): void;
  onTransportError(cb: (evt: TransportErrorEvent) => void): void;
}

// 实现：UnixDomainSocketTransport implements Transport
```

**使用语义**：

- listen 失败 reject（如 socket 占用 / 权限拒）；close 幂等
- 单连接失败：broadcast 经返回值 `{ failed: BroadcastFailure[] }` 暴露 / Gateway 按列表 dropConnection
- onDisconnect 透传 reason（单连接 'error' 事件原因 / 'close' 时携带）
- 不可预期失败（回调抛错 / server-level error）通过 `onTransportError` 显式通道（递归边界 = console.error 兜底 / 类 `[AUDIT CRITICAL]` 形态 / 详 `l1_transport.md` §5）
- 实时通道，掉线即丢（持久通信走 L2 Messaging 的 inbox 加 outbox）
- 重连策略调用方自治
- 广播 best-effort，不保证全部送达

**归本模块**：clawforum 持久双向连接通道能力的唯一入口。业务模块要做实时双向通信必经本模块。

**不归本模块**：

- 业务消息语义（消息 schema、路由、内容含义），归各调用方
- 跨进程消息持久化（实时通道掉线即丢），归 L2 Messaging（inbox 加 outbox）
- 重连策略，调用方自治

**不可消除耦合理由**：M#3 资源唯一归属（持久双向连接是单一资源）加 M#5 业务模块不允许直接 import 具体协议库。Design Principle「clawforum 支持分布式部署和跨 OS 平台」加「CLI 是 claw 和 motion 的唯一对外入口，分布式场景下可替换为远程调用」derive 必经本模块抽象（协议异构吸收）。

## FileWatcher [capability, direct + DI]

**生产方**：`l1_file_watcher`

**消费方**：所有需要订阅文件变化的模块（如 L4 AsyncTaskSystem 订阅 tasks/pending/、L4 ContractSystem 订阅 contract/active/ 等）

**接口签名**：

```ts
export type WatcherErrorContext = 'watch' | 'callback' | 'ready';

export type WatchEventType = 'add' | 'change' | 'unlink' | 'addDir' | 'unlinkDir';

export interface WatchEvent {
  type: WatchEventType;
  path: string;
  stats?: { size: number; mtime: Date };
}

export interface Watcher {
  close(): Promise<void>;
  isActive(): boolean;
  getPath(): string;
}

export function createWatcher(
  absolutePath: string,
  callback: (event: WatchEvent) => void,
  options?: {
    recursive?: boolean;
    ignored?: ReadonlyArray<string | RegExp>;
    stability?: 'stable' | 'immediate';
    persistent?: boolean;
    onReady?: () => void;
    onError?: (err: Error, context: WatcherErrorContext) => void;
    /**
     * Fallback poll 间隔（ms）/ override default 500ms
     * 仅 `stability === 'immediate'` + `process.platform === 'darwin'` 双条件下启用
     * 其他场景（'stable' 模式 / Linux / Windows）忽略此字段
     * 默认值 = 500ms（phase352 实测验证）
     * 当前 0 caller 用 override / 应然 escape hatch 预留
     */
    fallbackPollMs?: number;
  },
): Watcher;
```

**使用语义**：

- `createWatcher` 工厂式调用 / 不暴露 class instance
- options 为 inline 字面量类型 / 不立独立 `WatcherOptions` interface（caller 不持 options 实例 / 一次性传入）
- 运行期错误经 `onError(err, context)` 单回调传给调用方 + `WatcherErrorContext` discriminator 标识来源（'watch' = chokidar 错 / 'callback' = 主回调抛错 / 'ready' = onReady 抛错）
- callback / onReady / onError 抛错经 try/catch 隔离 / 不冒泡到 chokidar
- 调用方提供路径，本模块不知业务目录语义
- watch 触发后读文件内容由调用方做（用 L1 FileSystem）
- 错误审计调用方按自己业务上下文做（事件命名空间归调用方 / 4 caller 各自有 prefix / 详 `l1_file_watcher.md` §5）
- `close()` 幂等 / `isActive()` 状态查询 / `getPath()` 返绝对路径
- **fallback poll 内置（per phase 469 G1-G3 derive）**：仅 `stability === 'immediate'` + macOS 平台双条件自动启用 fallback poll setInterval / 默认 500ms / 触发时 emit `{type:'change', path: absolutePath}` event 给 callback / caller 不感知 fallback 存在 / `close()` 内 clearInterval 兜底 / 详 modules/l1_file_watcher.md §1.做 + `coding plan/phase469/derive.md`

**归本模块**：clawforum 文件变化通知能力的唯一入口。业务模块要订阅文件变化必经本模块。

**不归本模块**：

- 业务目录语义（具体业务目录），调用方提供路径
- 内容读取（watch 触发后读文件内容），归调用方（用 L1 FileSystem）
- 错误审计加事件命名空间，调用方按自己业务上下文做

**不可消除耦合理由**：M#3 资源唯一归属（OS 文件系统变化通知机制是单一资源）加 M#5 业务模块不允许直接 import 文件 watch 库。Design Principle「clawforum 支持分布式部署和跨 OS 平台」derive 必经本模块抽象（OS 通知机制异构吸收）。

> **应然立场已 sharpen done（phase 469 / design only）**：FileWatcher 内置 OS watch 不可靠兜底（如 macOS FSEvents 静默停火 → fallback poll）是 OS 抽象抹平范围一部分（per modules/l1_file_watcher.md §1）/ 不 leak 到 L2 业务模块。createWatcher options 加 `fallbackPollMs?: number` 字段（详上接口签名）/ 实现内部综合判定 `enableFallback = stability === 'immediate' && process.platform === 'darwin'`。**code 实施推 phase 469 Step 3-5（用户实施 / 详 `coding plan/phase469/执行计划.md`）**。

**Design-gap L1.G1-G3 closed by phase 469 derive（design only）**：

| Gap | 业务概念 | derive 结论（详 `coding plan/phase469/derive.md`）|
